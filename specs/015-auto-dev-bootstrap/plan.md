# Implementation Plan: Automatic Development Bootstrap

**Branch**: `015-auto-dev-bootstrap` | **Date**: 2026-07-18 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/015-auto-dev-bootstrap/spec.md`

## Summary

Make the existing development entry point verify that the repository-local Vite CLI is available before port selection. When it is absent, a small dependency-bootstrap module performs a locked `npm ci`, validates the resulting tool, and only then returns control to normal Vite startup. Dependency orchestration is injected and covered by a standalone Node check so failure paths never alter the real installation during tests.

## Technical Context

**Language/Version**: JavaScript ES modules on the project's supported Node.js runtime; TypeScript 5.6 remains unchanged for application code

**Primary Dependencies**: Node.js built-in `child_process`, `fs`, `path`, and `url`; npm CLI; existing Vite 5 development dependency

**Storage**: Local `package-lock.json` and `node_modules`; no application data or persistent feature state

**Testing**: Node.js `assert` check scripts for dependency orchestration and existing port selection; full Vitest suite; TypeScript/Vite production build

**Target Platform**: Windows desktop development as primary; compatible Node.js development environments as secondary

**Project Type**: Electron desktop application with repository-local development scripts

**Performance Goals**: Less than 100 ms added startup overhead when Vite is already installed; no network or package-manager process on repeat startup

**Constraints**: Preserve deterministic lockfile installation, inherited installer output, existing Vite arguments and automatic port selection; do not modify build/test/dist dependency behavior

**Scale/Scope**: One development entry point, one bootstrap module, one focused check script, package script registration, and Getting Started documentation
